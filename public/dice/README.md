# Production Dice Assets

This folder contains production-ready custom dice assets for the Daisu app.

## Folder Structure

```
dice/
├── README.md (this file)
├── manifest.json (auto-generated list of all sets)
└── [set-name]/
    ├── set.json              # Set metadata (required)
    └── [dice-name]/
        ├── model.glb         # 3D model (required)
        ├── metadata.json     # Dice configuration (required)
        └── thumbnail.png     # Preview image (optional, 256x256)
```

## Set Metadata (`set.json`)

Each set folder must have a `set.json` file:

```json
{
  "id": "fantasy-set",
  "name": "Fantasy Collection",
  "artist": "ArtistName",
  "description": "A collection of fantasy-themed dice",
  "releaseDate": "2024-12-07",
  "tags": ["fantasy", "dragons"],
  "availability": "always"
}
```

## Dice Metadata (`metadata.json`)

Each dice folder has a `metadata.json` with full configuration:

```json
{
  "version": "1.0",
  "diceType": "d20",
  "name": "Dragon D20",
  "artist": "ArtistName",
  "created": "2024-12-07",
  "scale": 1.0,

  "rarity": "legendary",
  "description": "A fierce dragon-themed d20",
  "tags": ["dragon", "fire"],

  "faceNormals": [...],
  "physics": { "density": 0.3, "restitution": 0.3, "friction": 0.6 },
  "colliderType": "hull",
  "colliderArgs": {},

  "animations": []
}
```

## New Fields in Dice Metadata

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `rarity` | string | Yes | common, uncommon, rare, epic, legendary, mythic |
| `description` | string | No | Flavor text for the dice |
| `tags` | string[] | No | Tags for filtering/search |

## Creating Dice Assets

1. Use the **Artist Testing Platform** in the app to:
   - Upload your GLB model
   - Configure scale, physics, and face mappings
   - Set rarity and description
   - Generate and download the `metadata.json`

2. Create the folder structure:
   ```
   dice/
   └── my-set/
       ├── set.json
       └── my-dice/
           ├── model.glb
           └── metadata.json
   ```

3. Run `npm run generate-dice-manifest` to update the manifest

4. Rebuild the app

## Manifest Generation

The `manifest.json` file is auto-generated by scanning the folder structure.
It lists all available sets and dice for the loader to use.

Run: `npm run generate-dice-manifest`
